# First Tester Checklist

Use this checklist before sending StanleySync to an outside tester.

## Access

- [ ] `http://localhost:3000/login` loads.
- [ ] `demo@stanleysync.app` logs in with the local demo password.
- [ ] Demo User lands in StanleySync Demo.
- [ ] Demo User does not see the Product mode / workspace switcher.
- [ ] Demo User cannot access `/admin/settings/users`.
- [ ] Demo User cannot access StanleySync Labs / CalOps pages.

## Quote Workflow

- [ ] Public quote intake at `/quote` loads.
- [ ] Quote intake questions are clear and not repetitive.
- [ ] Submitted quote appears in the demo workspace quote list.
- [ ] Quote detail opens.
- [ ] Quote can convert to a job.

## Job Workflow

- [ ] Job list opens.
- [ ] Job detail opens.
- [ ] Job status can update.
- [ ] Work order PDF exports.

## Invoice Workflow

- [ ] Invoice can be created from quote or job.
- [ ] Invoice detail opens.
- [ ] Payment link can be added by owner/admin.
- [ ] Quote PDF exports.
- [ ] Invoice PDF exports.

## Owner Controls

- [ ] `owner@stanleysync.app` can log in.
- [ ] System Owner sees Product mode selector.
- [ ] Product mode selector shows StanleySync App, StanleySync Labs, and StanleySync Demo.
- [ ] System Owner can open `/admin/settings/users`.
- [ ] System Owner can add, edit, assign workspace, and deactivate users.
- [ ] System Owner can access StanleySync Labs.

## Final Notes

- [ ] Known limitations were explained to the tester.
- [ ] Feedback form was sent.
- [ ] Tester knows who to contact with screenshots or issues.
