# Web Demo Launch Checklist

Use this checklist before sending StanleySync App to beta testers.

## Hosted Environment

- [ ] Fresh hosted database is created.
- [ ] Environment variables are set.
- [ ] Demo mode is enabled.
- [ ] Pilot mode is enabled if using pilot controls.
- [ ] Labs module is disabled or restricted.
- [ ] Public signup is disabled unless intentionally enabled.
- [ ] Demo users are seeded.
- [ ] Demo user is locked to StanleySync Demo.

## Deployment

- [ ] App is deployed.
- [ ] `/` loads.
- [ ] `/demo/start` loads.
- [ ] `/quote` works.
- [ ] `/login` works.
- [ ] Dashboard loads after login.
- [ ] Demo user cannot switch workspaces.
- [ ] Demo user cannot access system settings.
- [ ] Demo user cannot access StanleySync Labs.

## Workflow

- [ ] Quote can be submitted.
- [ ] Quote appears in admin.
- [ ] Quote can convert to job.
- [ ] Job can be updated.
- [ ] Invoice can be created.
- [ ] Quote PDF exports.
- [ ] Work order PDF exports.
- [ ] Invoice PDF exports.
- [ ] Feedback page works.

## Handoff

- [ ] Tester has demo link.
- [ ] Tester has login email.
- [ ] Tester has password.
- [ ] Tester has feedback instructions.
- [ ] Customer-facing docs do not include local setup instructions.
